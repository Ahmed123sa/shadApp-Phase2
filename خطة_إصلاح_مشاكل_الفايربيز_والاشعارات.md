# خطة إصلاح مشاكل Firebase والإشعارات

**تاريخ الخطة:** 6 يوليو 2026
**المشروع:** Ahmed123sa/shadApp-Phase2

---

## 1. ملف iOS Firebase (`GoogleService-Info.plist`)

**السبب:** الملف مش موجود في `mobile/ios/Runner/` مع إن `android/app/google-services.json` موجود.

**المطلوب:**
- ينزل الملف من Firebase Console -> Project settings -> Add app -> iOS
- يوضع في `mobile/ios/Runner/GoogleService-Info.plist`
- يتأكد إنه مضاف لـ Xcode target (Copy Bundle Resources)
- تفعيل Push Notifications capability + Background Modes في Xcode
- رفع APNs key في Firebase Console → Cloud Messaging → Apple app config

**ملاحظة:** محتاج Apple Developer account.

---

## 2. الإشعارات تعمل بشكل غير متزامن (عدم استخدام ShouldQueue)

**السبب:** كل 12 Notification class عندهم `use Queueable;` لكن مش `implements ShouldQueue`.

**الحل:** إضافة `implements ShouldQueue` لكل Notification class:
- `ApprovalRequestedNotification`
- `ApprovalRespondedNotification`
- `ChatMessageSentNotification`
- `ContractClientApprovedNotification`
- `ContractClientSignedNotification`
- `ContractCompanyApprovedNotification`
- `ContractCompletedNotification`
- `ContractReminderNotification`
- `ContractSentNotification`
- `MeetingReminderNotification`
- `PaymentCreatedNotification`
- `PaymentReviewedNotification`

**أيضًا:**
- التأكد من `QUEUE_CONNECTION=database` في `.env`
- تشغيل `php artisan queue:work --tries=3 --backoff=10` على السيرفر

## 3. عدم تنظيف التوكنات الملغاة في FcmChannel

**السبب:** `FcmChannel::send()` بيستخدم `pluck('token')` — بيرجع array of strings مش Models، مفيش check على `unregistered` ولا delete. `FirebaseService::sendToUser()` عنده المنطق الصحيح لكن مش مستخدم.

**الحل:** تعديل `backend/app/Notifications/FcmChannel.php`:
- تغيير `pluck('token')` → `get()` عشان نجيب Models
- إضافة check بعد `sendMessage()`: لو `$result === 'unregistered'` نحذف التوكن (`$deviceToken->delete()`)

---

## 4. تذكيرات العقود/الاجتماعات مش مفعّلة

**السبب:** `ContractReminderNotification` و `MeetingReminderNotification` موجودين لكن مفيش حاجة تستدعيهم — ولا `Schedule::` في `routes/console.php`.

**الحل:**
- إنشاء `app/Console/Commands/SendContractReminders.php`
- إنشاء `app/Console/Commands/SendMeetingReminders.php`
- جدولتهم في `routes/console.php`:
  - `contracts:send-reminders` → `dailyAt('09:00')`
  - `meetings:send-reminders` → `everyThirtyMinutes()`
- إضافة `* * * * * cd /path && php artisan schedule:run` في Cron السيرفر

---

## 5. عدم تطابق مسار Broadcasting Auth

**السبب:** `Broadcast::routes(['middleware' => ['auth.any:sanctum,client']])` بدون `prefix => 'api'`. لكن `echo.ts` بيستخدم `NEXT_PUBLIC_API_URL + '/broadcasting/auth'` و `NEXT_PUBLIC_API_URL` غالبًا `.../api`.

**الحل:** إضافة `prefix => 'api'` في `BroadcastServiceProvider.php`:
```php
Broadcast::routes([
    'prefix' => 'api',
    'middleware' => ['auth.any:sanctum,client'],
]);
```
